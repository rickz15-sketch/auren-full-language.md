# AUREN LANGUAGE — OFFICIAL SPECIFICATION (v3.0)

## 1. PURPOSE

This document defines the official structural rules of the Auren language.

It establishes:
- grammatical constraints
- semantic behavior
- compatibility rules
- evolution boundaries

Auren is a structured, evolving language system, but this specification defines its **stable core layer**.

---

## 2. LANGUAGE TYPE

Auren is:

- Constructed (artificial)
- Symbolic
- Evolving
- Context-sensitive
- Multi-interpretative

It is NOT:

- a natural human language
- a fixed programming language
- a closed system

---

## 3. CORE SENTENCE STRUCTURE

Default structure:

VERB + AGENT + OBJECT

Example:
dor ara nora  
("speak I language")

---

## 4. ALLOWED VARIATIONS

Auren allows three structural modes:

### 4.1 Standard Mode
VERB + AGENT + OBJECT

### 4.2 Emphasis Mode
AGENT + VERB + OBJECT

### 4.3 Poetic Mode
OBJECT + VERB + AGENT

Meaning shifts depending on mode.

---

## 5. CORE LEXICAL ROOTS (IMMUTABLE)

The following roots are stable and SHOULD NOT be deleted:

- ara = self
- eri = other
- nora = language
- vesh = thought
- thal = knowledge
- lumen = truth
- nox = falsehood
- dor = speak
- cre = create
- ent = understand

---

## 6. MORPHOLOGICAL RULES

### 6.1 Time suffixes

- -a = present
- -en = past
- -ir = future
- -or = abstract/timeless

---

### 6.2 Negation

Prefix:
nor-

Example:
nor-dor = not speak

---

### 6.3 Question marker

Prefix:
ke-

Example:
ke dor ara nora?

---

## 7. SEMANTIC BEHAVIOR

Meaning in Auren is determined by:

1. structure
2. context
3. repetition
4. evolution history

Words do NOT have single fixed meanings.

---

## 8. AMBIGUITY RULE

Ambiguity is valid when:

- it increases expressive depth
- it allows multiple interpretations
- it is structurally consistent

Ambiguity is NOT considered error.

---

## 9. EVOLUTION RULE (CONTROLLED)

Auren may evolve only if:

- new usage becomes consistent
- changes do not break core roots
- grammar remains interpretable

Core roots listed in section 5 are stable.

---

## 10. COMPATIBILITY RULE

All future versions of Auren MUST:

- preserve core roots
- maintain sentence structure compatibility
- remain partially interpretable from previous versions

---

## 11. META PRINCIPLE

> Auren is not defined by its vocabulary.
> Auren is defined by its rules of transformation.

---

## 12. VERSION

Current specification version: v3.0

END OF SPECIFICATION
