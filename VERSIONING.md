# AUREN LANGUAGE — VERSIONING SYSTEM

## 1. PURPOSE

This document defines how Auren evolves across versions.

---

## 2. VERSION FORMAT

vMAJOR.MINOR.PATCH

Example:
v3.0.0

---

## 3. VERSION MEANING

### MAJOR
Structural changes (grammar/system overhaul)

### MINOR
New words, expansions, features

### PATCH
Fixes and clarifications

---

## 4. COMPATIBILITY RULE

All versions MUST:

- preserve core vocabulary
- maintain sentence structure compatibility
- remain partially readable backwards

---

## 5. EVOLUTION RULE

Changes are only valid if:

- they improve expressiveness
- they remain consistent
- they do not break core structure

---

## 6. BRANCHING RULE

If changes diverge too much:

→ create dialect instead of replacing system

---

## 7. STABLE CORE

Never change:

- core grammar structure
- core roots
- compatibility system

---

END OF VERSIONING SYSTEM
